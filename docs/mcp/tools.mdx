## `create_project`

Permission: `manage_projects`

| Param | Type | Required | Default |
|-------|------|----------|---------|
| `slug` | string | ✓ | |
| `name` | string | ✓ | |
| `description` | string | | |
| `budget` | number (≥0) | | |
| `currency` | string | | `USD` |
| `environment` | `development` \| `staging` \| `production` \| `other` | | `production` |

Creates a project explicitly with name, budget, and metadata. Returns `409` if the slug already exists (unlike implicit creation via `log_usage`, which upserts silently).

**Local:** `POST /api/v1/projects`.  
**Remote:** DB insert via service role.

## `list_projects`

Permission: `read_summaries`

| Param | Type | Required | Default |
|-------|------|----------|---------|
| `include_archived` | boolean | | `false` |

Returns all projects visible to the API key (respects project scope conditions).

**Local:** `GET /api/v1/projects`.  
**Remote:** DB query via service role.

## `get_monthly_summary` (remote only)

Permission: `read_summaries`  
No parameters. Same shape as `GET /api/v1/summary/month`.

```json
{
  "period": "month",
  "total_usd": 0,
  "usage_usd": 0,
  "subscription_usd": 0,
  "expense_usd": 0,
  "by_project": {},
  "top_project": null,
  "message_count": 0
}
```
